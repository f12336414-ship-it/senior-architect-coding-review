# ADR 模板

只为会长期影响边界、数据、基础设施、兼容性或运行风险的决策创建 ADR。

```markdown
# ADR-NNN: 决策标题

日期：YYYY-MM-DD
状态：Proposed | Accepted | Superseded | Rejected
所有者：团队或角色

## 背景

描述业务问题、当前事实、约束和必须保持的不变量。

## 决策

一句话说明选择，再描述关键边界、数据流和运行语义。

## 候选方案

### 方案 A

- 收益：
- 成本：
- 风险：
- 可逆性：

### 方案 B

- 收益：
- 成本：
- 风险：
- 可逆性：

## 结果

- 正面影响：
- 负面影响：
- 新增运维责任：
- 安全、隐私和合规影响：

## 迁移与回滚

说明发布顺序、兼容窗口、数据校验、停止条件和恢复方案。

## 验证

列出测试、指标、告警和成功阈值。

## 重新评估条件

列出会使该决策失效的可观察信号。
```

ADR 记录“为什么”，不要复制实现细节。决策被替代时保留原文并链接新 ADR。
