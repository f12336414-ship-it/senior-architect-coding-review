# 可靠性与安全

## 外部依赖

- 为每个网络调用设置与端到端预算一致的超时。
- 只对暂时性失败重试；限制次数、使用退避与抖动，并遵守总截止时间。
- 只有操作幂等或具有幂等键时才自动重试写操作。
- 限制并发、队列长度和请求体大小，避免无界资源消耗。
- 使用隔离、熔断、降级或负载丢弃阻止级联故障。
- 明确取消信号是否传播，以及资源如何清理。

不要把熔断器当成普遍装饰。只有依赖故障会持续并需要快速失败时使用。

## 容量与背压

- 找出最慢资源和不可水平扩展的瓶颈。
- 区分平均值、尾延迟、突发流量和恢复流量。
- 让生产者感知消费者容量；不要无限缓存积压。
- 为批任务、后台任务和在线流量设置独立预算或隔离。
- 缓存必须定义失效、陈旧容忍、击穿保护和权威来源。

## 可观测性

为关键路径提供：

- 业务成功、拒绝和失败指标。
- 延迟、错误、流量、饱和度和队列年龄。
- 可关联请求、任务和消息的结构化日志或追踪标识。
- 面向用户影响的告警，而不是只对内部异常计数告警。
- 部署版本、配置和特性开关维度。

日志不得泄漏秘密、令牌、密码或不必要的个人数据。

## 安全边界

- 区分认证、授权和业务资格检查。
- 在服务端和资源所有者边界执行授权，不信任客户端隐藏字段。
- 所有查询和命令考虑租户隔离、对象级越权和批量接口越权。
- 对输入设置类型、大小、格式和语义限制，并安全处理输出。
- 使用参数化查询和安全解析器；避免拼接命令、路径、模板或查询。
- 秘密只通过受控存储和短期凭证传递，不写入仓库、日志或错误响应。
- 高风险操作记录不可抵赖的审计信息，但不记录敏感内容本身。
- 依赖升级同时检查漏洞、许可证、供应链来源和行为兼容性。

## 威胁快速扫描

检查攻击者能否：

- 冒充其他用户、租户或内部服务。
- 修改不属于自己的对象或绕过状态转换。
- 重放请求、制造并发竞争或扩大重试。
- 注入代码、查询、路径、模板或反序列化载荷。
- 利用错误、日志、时间差或元数据获取敏感信息。
- 通过大输入、高基数、慢查询或任务洪泛耗尽资源。

安全问题优先使用消除危险状态和集中强制边界的方案，不依赖开发者记住每个调用点。
